# @flowchestra/prisma-guarddog-importer-postgres

Read existing `pg_policies` and `information_schema.column_privileges` from a live Postgres database and render a TypeScript scaffold the user can fold into their guarddog schema.

**Scaffold-only**, per [ADR-0012](../../docs/adr/0012-scaffold-only-importer.md). Existing policies are emitted as `rawSql()` + `.todo()` — never as typed predicates. The importer surfaces evidence; the user authors intent.

## What lives here

- **`readPgPolicies(client, opts?)`** — query `pg_policies` for the configured schema and return normalized `ImportedPolicyRow[]` (includes the policy SQL — `usingExpression`, `withCheckExpression` — for scaffold generation).
- **`readPolicyInventory(client, opts?)`** — lighter-weight catalog inventory: one row per live policy with `permissive: boolean` and the `pg_description` comment. Consumed by `computePolicyDrift` in the CLI to classify foreign / managed / acknowledged ([ADR-0029](../../docs/adr/0029-handling-existing-rls-policies.md)).
- **`readColumnPrivileges(client, opts?)`** — query `information_schema.column_privileges` (filtered to SELECT / INSERT / UPDATE) and return `ImportedColumnPrivilege[]`.
- **`generateScaffold({ policies, columnPrivileges, modelMap?, banner? })`** — render a complete `.ts` module exporting an `importedPolicies(guard)` function.

The reader accepts any `{ query(text, params): Promise<{rows}> }` shape — pass `pg.Client`, `pg.Pool`, `pg.PoolClient`, or a custom adapter. No hard dependency on the `pg` package.

## Install

```sh
pnpm add -D @flowchestra/prisma-guarddog-importer-postgres pg
```

## 30-second usage

```ts
import { generateScaffold, readColumnPrivileges, readPgPolicies } from '@flowchestra/prisma-guarddog-importer-postgres'
import { writeFile } from 'node:fs/promises'
import { Client } from 'pg'

const client = new Client({ connectionString: process.env.DATABASE_URL })
await client.connect()

const policies = await readPgPolicies(client)
const columnPrivileges = await readColumnPrivileges(client)

const source = generateScaffold({
  policies,
  columnPrivileges,
  modelMap: { workspace: 'Workspace', workbench: 'Workbench' }, // table → Prisma model
})

await writeFile('prisma/scaffold.generated.ts', source, 'utf8')
await client.end()
```

Then review the generated file, replace each `rawSql()` with a typed predicate, and remove the `.todo()` markers as you go.

## Output shape

```ts
// Auto-generated by `@flowchestra/prisma-guarddog-importer-postgres`.
// Review each rawSql() and replace with a typed predicate; remove .todo() as you go.

import type { Guarddog } from 'prisma-guarddog'

export function importedPolicies(guard: Guarddog): void {
  guard.model('Workbench').policy('app_user')
    .rawSql('select', "tenant_id = current_setting('request.jwt.claims', true)::json->>'tenantId'::uuid")
    .todo("replace raw SQL with typed predicate — imported from pg_policies \"workbench_app_user_select\"")

  guard.model('File').columnPrivileges({
    "apiKey": { select: ['app_system'], update: ['app_system'] },
  })
}
```

## Heuristic table → model resolution

Tables not present in `modelMap` fall through to a `snake_case_table` → `SnakeCaseTable` heuristic, and the generated scaffold flags those with a TODO. Provide an explicit `modelMap` (often built from your Prisma DMMF) to skip the heuristic.
